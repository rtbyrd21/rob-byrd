Kroger's microsite CMS was created to provide recipe content to all of Kroger's microsites. In order to work across multiple sites it has been created to be extremely flexible and dynamic. The user interface has undergone multiple rounds of iterations and user-testing.

Context

I came on-board Kroger's microsite team during their second site build. The previous site was required a 14,000 line JSON file which had been entered by hand. The microsite CMS was created to easily build, maintain and update recipes in JSON format for each new microsite. Tabs and fields are dynamilcally added, dropped and validated based on various rules for each microsite.

Scope

The CMS is an ongoing poject. The CMS was initially created with a very tight deadline. Modifications and dynamic updates to the UI are continually being added. With each user adding content into the CMS, feedback is alwats evaluated and iterated on to create a better user-experience.

Responsibilites
